◆ updateRobotState

void RobotProxy::updateRobotState ( )
slot

a slot of mTimerRobotStateChange timeout interupt, 从机械臂中读取信息，更新机械臂状态, started from connectRobot()

it read data from tcpDevice first, then change the byte order to local machine order. and call RobotProxy::_setRobotState() to set the readed data into RobotProxy::mRobotStateStream. after it, if force sensor is open, it read the force sensor(force sensor not read because the called function not used) then emit robot state change signal with robot data readed above to RobotSystemControl::updateRobotStateSlot(). in that func, state data will send to servo algrithm to advanced process.

Warning
the if sentense only consider the read success condition, when reading failed, QIODevice::read() returns -1. and resize() may use para 0;the function mainly deal with byte order problem and it can be dealed with using QDataStream, which used for it. the byte length of UR3 is 1108, UR5 is 1160.
Todo:
change the if sentence condition to bytes_read >= 0, and add resize() condition judge. using QDataStram to substitute complex code.
